Part 2: Connecting to & Preparing Your Server

With the server running, it's time to log in and install Docker, the software that will run your application.

1

Connect via SSH

Use your terminal and the `.pem` key file you downloaded to connect. First, make sure you set the correct permissions for the key file.

# Set permissions for your key
chmod 400 your-key-name.pem

# Connect to the server (replace with your key name and IP)
ssh -i "your-key-name.pem" ubuntu@your_public_ip_address
2

Install Docker

Run these commands on your server to install Docker and configure permissions. You will need to log out and back in after the final command.

# Update package lists
sudo apt-get update

# Install Docker
sudo apt-get install -y docker.io

# Add current user to the docker group
sudo usermod -aG docker ${USER}

# Log out to apply changes
exit

Part 3: Setting Up Automated Deployment

Now for the automation. We will set up a connection between GitHub and your EC2 server. This allows GitHub to send commands to your server to update the application whenever you push new code.

1

Create Deploy Key

First, generate an SSH key on your EC2 server. This key will give GitHub read-only access to your repository.

# Run this ON YOUR EC2 SERVER
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

Next, display the public key and copy its contents.

# Run this ON YOUR EC2 SERVER
cat ~/.ssh/id_rsa.pub
2

Clone Project on Server

Pull your code onto the server for the first time. This confirms the deploy key is working.

# Run this ON YOUR EC2 SERVER
git clone git@github.com:your-username/your-repo-name.git
3

Create GitHub Actions Workflow

On your local machine, create a file at .github/workflows/deploy.yml and paste this content. This is the recipe for your automation.

name: Build and Deploy to EC2
on:
  push:
    branches:
      - main
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: SSH into EC2 and Deploy
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.EC2_HOST }}
          username: ${{ secrets.EC2_USERNAME }}
          key: ${{ secrets.EC2_PRIVATE_KEY }}
          script: |
            cd your-repo-name
            git pull origin main
            docker build -t ai-smoke-tester .
            docker stop ai-app-container || true
            docker rm ai-app-container || true
            docker run -d --name ai-app-container -p 80:8080 ai-smoke-tester

Important: Update `your-repo-name` and the port mapping `-p 80:8080` in the script to match your project.

4

Set Up GitHub Secrets

In your GitHub repository under Settings > Secrets and variables > Actions, add the following secrets. This keeps your server credentials secure.

  • EC2_HOST: Your server's public IP address.
  • EC2_USERNAME: The username, which is `ubuntu`.
  • EC2_PRIVATE_KEY: The full content of the aws-key.pem file you downloaded from AWS.

Part 4: Your First Automated Deployment

You are all set! The final step is to push your new workflow file to GitHub, which will trigger the first automated deployment.

1

Push the Workflow File

On your local computer, commit and push the `deploy.yml` file to your GitHub repository.

git add .
git commit -m "Add CI/CD workflow for EC2 deployment"
git push origin main
2
